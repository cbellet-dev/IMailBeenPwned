> The modern, component-driven frontend dashboard for **IMailBeenPwned**.

## 📖 Overview

This is the presentation layer of the IMailBeenPwned project. It provides a clean, dark-themed, and responsive interface to query email addresses and beautifully display the resulting OSINT intelligence. It communicates directly with the Spring Boot Backend API to fetch security profiles.

## 🏗️ Architecture

The application has been explicitly structured to follow standard React modularity best practices, moving away from monolithic single-file implementations:

```text
src/
├── api/
│   └── client.js           # Fetch logic and backend Data formatting
├── components/             # Reusable, decoupled UI components
│   ├── Card.jsx            # Base card container
│   ├── DataRow.jsx         # Key-value visual row
│   ├── Modal.jsx           # Dark-themed overlay modal
│   ├── RiskMeter.jsx       # SVG-based circular risk indicator
│   ├── SectionLabel.jsx    # Typography component
│   ├── SourceDetail.jsx    # Accordion panel for OSINT sources
│   ├── SourcePill.jsx      # Colored source indicators 
│   ├── StatusBadge.jsx     # Visual Hit/Clean tags
│   └── Tag.jsx             # Micro-tags (e.g. "critical")
├── utils/
│   └── constants.js        # Global static variables and themes
├── EmailInvestigator.jsx   # Main orchestration view
└── main.jsx                # React Entrypoint
```

## ✨ Features

- **Component-Based UI:** Highly decoupled UI elements grouped under `src/components/`, ensuring easy scalability and maintenance.
- **AI Insight Integration:** Specifically designed UI cards and report headers that parse and highlight the context-aware veredict generated by Llama 3 on the backend.
- **Animated Risk Analysis:** A custom SVG gauge (`RiskMeter.jsx`) providing immediate visual feedback on the threat level.
- **Export Capabilities:** Instantly generate a consolidated `.txt` report containing the OSINT mapping and AI assessment.

## 🚀 Getting Started

1. **Install Dependencies:**
   ```bash
   npm install
   ```

2. **Run the Development Server:**
   ```bash
   npm run dev
   ```

3. **Production Build:**
   ```bash
   npm run build
   ```

*Ensure the backend is running on `localhost:8080` for the API calls in `src/api/client.js` to succeed.*
